1. Getting Started Back to TOC

BP Doctor is a native UE5 editor plugin that scans all Blueprints and Animation Blueprints in your project for 39 silent-failure patterns — from T-pose bugs to performance anti-patterns. It runs inside the editor as a dockable tab.

1.1 Interface Overview

Open BP Doctor from the editor menu: Window > BP Doctor (or Tools > BP Doctor).

The panel has three areas:

  • Toolbar — Scan, Fix All, Undo, Export, HTML, Checks, Settings, Clear + filters
  • Results List — All found issues with 7 columns: Severity, Confidence, Check, Type, Asset, Description, Fix
  • Detail Panel — Full information about the selected issue including fix instructions

1.2 Toolbar Reference

ButtonWhat It Does
Scan ProjectScans all Blueprints in /Game for issues
Fix AllOpens a checklist dialog to select and apply multiple fixes at once
Undo FixReverts the most recent fix via UE5's transaction system
ExportSaves a text report to a file you choose
HTMLExports a styled dark-themed HTML report
ChecksOpens the Check Library — browse and enable/disable all 39 checks
SettingsOpens the Settings dialog — experience mode, suppressions, custom rules, file paths
ClearClears all scan results

Right side of toolbar: Search box (filter by check code, asset name, or description), Experience Mode dropdown (Beginner/Intermediate/Expert), and severity checkboxes (Errors, Warnings, Info, Suppressed).

2. Scanning Your Project Back to TOC

Click Scan Project. BP Doctor loads every Blueprint and Animation Blueprint under /Game and runs all enabled checks against them. Progress is shown in the cyan progress bar.

2.1 Understanding Results

Each row in the results list shows:

ColumnMeaning
ERR / WRN / INFSeverity — how serious the issue is
HI / MED / LOConfidence — how reliable the detection is (hover for details)
Check codeWhich diagnostic rule was triggered (e.g., BP_SELF_CAST)
Anim / BPWhether this is an AnimBP or general Blueprint issue
AssetThe Blueprint asset where the issue was found
DescriptionWhat's wrong
FixYES = auto-fixable, - = manual fix required, DONE = already fixed

Rows are color-tinted by severity: red for errors, yellow for warnings, blue for info, green for fixed.

2.2 Health Grade

After scanning, your project gets a letter grade based on errors and warnings found:

GradeMeaning
A+0 errors, 0 warnings, 0 info — perfectly clean scan
A0 errors, 0 warnings (info-only is fine)
B+0 errors, 1-3 warnings
B0 errors, 4+ warnings
C1-2 errors
D3-5 errors
F6+ errors — silent failures shipping to prod

Errors are silent-failure bugs (T-pose, broken refs, etc.) that need fixing. Warnings are perf / architecture smells — context matters.

2.3 Confidence System

Each check has a confidence level that tells you how much to trust the result:

LevelMeaningWhat You Should Do
HIGHDeterministic detection. Near-zero false positives.Trust it. Fix it.
MEDIUMReliable heuristic. Rare false positives possible.Likely correct. Verify if unsure.
LOWHeuristic-based. May have false positives.Always verify in the editor before acting.

Hover over the confidence badge in the results list to see exactly how BP Doctor detected the issue. In Intermediate and Beginner modes, the detail panel also shows the detection method.

3. Fixing Issues Back to TOC

3.1 Auto-Fix

Issues marked YES in the Fix column can be fixed automatically:

  1. Select the issue in the results list
  2. Click Fix This Issue
  3. A preview dialog shows exactly what will change
  4. For some checks (blend weights, slot names): enter a custom value or leave blank for the default
  5. Click Apply Fix

Auto-fixable checks (7 total): BROKEN_BLEND_WT, MISSING_SLOT, DUP_SLOT, BLEND_WT_SUM, ORPHANED_NODE, BP_SELF_CAST, BP_DEBUG_NODES.

Profile note: 4 of 7 (BROKEN_BLEND_WT, MISSING_SLOT, DUP_SLOT, BLEND_WT_SUM) fire on the default Silent Failures Only profile. The other 3 (ORPHANED_NODE, BP_SELF_CAST, BP_DEBUG_NODES) are stylistic-tier checks — switch to Everything profile to surface them.

3.2 Manual Fix (Navigate)

For issues that can't be auto-fixed, click Navigate. This opens the Blueprint editor AND zooms directly to the problematic node. The detail panel shows step-by-step fix instructions under HOW TO FIX.

3.3 Reverting Fixes

Every fix is reversible:

  • Revert button — appears next to Fix This Issue when an issue has been fixed. Restores the Blueprint from a backup taken before the fix was applied.
  • Undo Fix (toolbar) — reverts the most recent fix using UE5's transaction system.

Backups are stored in Saved/BPDoctor/Backups/ and cleaned up after successful revert.

3.4 Fix All

Click Fix All to open a checklist showing every auto-fixable issue:

  • Each issue has a checkbox (all checked by default)
  • Uncheck any issues you want to skip
  • Use Select All / Deselect All buttons
  • Click Apply Selected Fixes

The entire batch is one undo step — a single Undo Fix reverts everything.

4. Check Reference (All 39 Checks) Back to TOC

4.1 AnimBP Checks (1-12, 27-39)

#1 NULL_ANIM_REF — Null Anim Reference ERROR HIGH

Sequence Player node has no animation asset assigned.

How to Fix:
1. Click Navigate to open the AnimBP
2. Find the Sequence Player node with no animation
3. In its Details panel, set the Sequence property to a valid AnimSequence
4. Compile and save the AnimBP
Detection: Scans all AnimGraphNode_SequencePlayer nodes. Flags any where the Sequence property is null.
#2 BROKEN_BLEND_WT — Broken Blend Weight WARN MED Auto-fixable

Blend weight value is outside the valid [0.0, 1.0] range.

How to Fix:
AUTO-FIX: Clamps the value to [0.0, 1.0]. Use the custom input field to set a specific value.

MANUAL: Navigate to the LayeredBoneBlend node > find the BlendWeight pin > set a value between 0.0 and 1.0.
Detection: Reads default pin values on LayeredBoneBlend nodes. Flags any BlendWeight pin with a value outside [0.0, 1.0].
#3 SKEL_MISMATCH — Skeleton Mismatch ERROR MED

Your AnimBP is set to one skeleton (e.g. UE5 Mannequin) but one of its animations was made for a different skeleton (e.g. Mixamo or an older UE4 rig). The animation won't play correctly — bones end up in wrong positions or the asset fails to cook.

How to Fix (easy — swap the animation):
1. Click Navigate to open the AnimBP
2. Find the animation node with the yellow warning badge
3. In the Details panel on the right, find the Sequence or Anim Sequence property
4. Click the dropdown and pick an animation built for the SAME skeleton your AnimBP uses
5. Hit Compile (top-left)

To check your skeleton: Open the AnimBP > Class Defaults tab (top-right) > Target Skeleton field shows the name.

How to Fix (harder — retarget):
1. Window menu > Retarget Manager
2. Source Skeleton = the animation's original skeleton
3. Target Skeleton = your AnimBP's skeleton
4. Map the bones (UE auto-maps most common rigs)
5. Click Retarget — creates new animations on your skeleton
6. Swap the AnimBP node to the new retargeted animation

Why it happens: common cause is importing Mixamo anims or FBX from another project without retargeting first.
Detection: Compares the AnimBP's TargetSkeleton against each referenced animation's Skeleton.
#4 MISSING_SLOT — Missing Default Slot WARN MED Auto-fixable

AnimBP references montages but has no Slot node in the AnimGraph — montages will not play.

How to Fix:
AUTO-FIX: Adds a DefaultSlot node to the AnimGraph. After fixing, open the AnimBP and connect the Slot node into your pose chain.

MANUAL:
1. Open the AnimBP > AnimGraph
2. Right-click > Add Slot node (search 'Slot')
3. Set the Slot Name (usually 'DefaultSlot')
4. Connect it between your pose chain and the Output Pose
5. Make sure your Montages use the same Slot Name
Detection: Searches AnimGraph for AnimGraphNode_Slot. Only flags if montage references are present. MEDIUM confidence — montages may be triggered from other BPs.
#5 BROKEN_TRANS — Broken Transition WARN MED

State machine contains unreachable states with no inbound transitions.

How to Fix:
1. Open the State Machine graph
2. Look for states with no arrows pointing INTO them
3. Either add a transition from another state, or delete the unreachable state
4. Verify every state has at least one inbound transition from Entry
Detection: Counts states vs transitions. Heuristic: if states > transitions + 1, some states may be unreachable.
#6 TPOSE_FALLBACK — T-Pose Fallback ERROR MED

Your character will T-pose (arms out like a scarecrow) the moment this blend node runs. One of the most common silent-failure bugs in shipped games.

What's happening: A Layered Blend Per Bone node mixes two poses — a base pose (e.g. idle) and a layer pose (e.g. upper-body aim). If the BasePose input has no connection, the node has nothing to blend FROM, so it outputs the skeleton's rest pose — T-pose.

Visual clue: The BasePose input is the FIRST input pin on the LayeredBoneBlend, top-left of the node. No white wire going into it = bug.

How to Fix:
1. Click Navigate to open the AnimGraph
2. Find the flagged Layered Blend Per Bone node (yellow/red badge)
3. Look at the top-left input pin labeled Base Pose
4. Find your main pose chain (usually a State Machine output or a BlendSpace)
5. Drag a wire from its output INTO the Base Pose pin
6. Hit Compile
7. Press Play — character should no longer T-pose

If the node shouldn't exist: select it, press Delete, reconnect the surrounding nodes so the pose chain is continuous from the State Machine down to Output Pose.

Why it's subtle: UE's compiler does NOT warn about disconnected BasePose inputs. The AnimBP compiles clean. You only see the bug at runtime — which is why this ships to prod constantly.
Detection: Checks LayeredBoneBlend nodes for disconnected BasePose input pins.
#7 ORPHANED_NODE — Orphaned Node INFO MED Auto-fixable

Node exists in the graph but is not reachable from the Output Pose.

How to Fix:
AUTO-FIX: Deletes all AnimGraph nodes not reachable from the Output Pose root. Only affects top-level graph nodes — state machine contents are preserved.

MANUAL:
1. Click Navigate to find the orphaned node
2. Check if it was accidentally disconnected (reconnect if needed)
3. If truly unused, select it and press Delete
4. Clean up any connected but also-orphaned nodes
Detection: Skips AnimBPs with 15 or fewer anim nodes. Walks from Output Pose root via input pins. Flags if more than 3 nodes are unreachable.
#8 INVALID_BSPACE — Invalid BlendSpace WARN MED

BlendSpace asset has 0 or 1 sample points — cannot interpolate.

How to Fix:
1. Open the BlendSpace asset
2. Add sample points by dragging animations onto the grid
3. Place at least 2 samples at different axis positions
4. Preview the interpolation by moving the green crosshair
#9 MISSING_NOTIFY — Missing Notify INFO MED

AnimNotify references a function or event that has been deleted.

How to Fix:
1. Open the AnimBP > Event Graph
2. Check if the notify handler exists (AnimNotify_[Name])
3. If deleted: right-click > Add AnimNotify Event
4. Reconnect to gameplay logic (sound, VFX, etc.)
#10 DUP_SLOT — Duplicate Slot Name WARN HIGH Auto-fixable

Same slot name used by multiple Slot nodes in the AnimGraph.

How to Fix:
AUTO-FIX: Renames duplicates with _2, _3 suffixes. Use custom input to set your own name.

MANUAL: Open AnimGraph > find Slot nodes > give each a unique name (e.g., UpperBody, LowerBody).
Detection: Compares SlotName on all Slot nodes. Exact string match — HIGH confidence.
#11 UNUSED_VAR — Unused Variable INFO LOW

AnimInstance variable declared but never read in the AnimGraph.

How to Fix:
1. Find the flagged variable in My Blueprint
2. Search for it (Ctrl+F)
3. If truly unused, delete it

NOTE: Verify in editor — this check uses heuristics and may have false positives.
Detection: Heuristic — counts properties vs VariableGet nodes. LOW confidence — may miss struct access patterns.
#12 DEPRECATED_NODE — Deprecated Node WARN MED

AnimGraph uses a node class marked deprecated.

How to Fix:
1. Navigate to the deprecated node
2. Hover to see the deprecation warning
3. Right-click > Find Replacement
4. Replace and reconnect pins
#27 MM_NO_DATABASE — MotionMatching Missing Database ERROR HIGH

MotionMatching node has no Pose Search Database assigned — outputs T-pose silently.

How to Fix:
1. Navigate to the MotionMatching node
2. In Details, assign a UPoseSearchDatabase to the Database property
3. Compile the AnimBP — the T-pose should be replaced by DB-selected poses
Detection: Scans every AnimGraphNode_MotionMatching; flags any with a null Database reference. HIGH confidence.
#28 MM_NO_INERTIALIZATION — MotionMatching Missing Inertialization WARN MED

MotionMatching output chain has no Inertialization node downstream — pose pops on every DB selection.

How to Fix:
1. Navigate to the MotionMatching node
2. Insert an Inertialization node between MM output and Output Pose (or the next blend node)
3. Set a reasonable blend time (0.15–0.3s is typical for locomotion)
Detection: From the MM node, walks downstream output connections; flags if no Inertialization node is found before Output Pose. MEDIUM confidence — intentional pop-cut styles may be false positives.
#29 SLOT_NAME_MISMATCH — Slot Name Not Registered in Skeleton WARN MED

Slot node uses a SlotName that isn't registered in the Skeleton's slot groups — montages targeting this slot will silently fail.

How to Fix:
1. Navigate to the Slot node, note its SlotName
2. Open the AnimBP's Target Skeleton
3. In the Skeleton editor: Anim Slot Manager > add the slot name to an existing group or create a new group
4. Save the skeleton
Detection: Reads Slot node SlotName; resolves the AnimBP's TargetSkeleton and checks its registered slot groups. MEDIUM confidence — heuristic-match for group names.
#30 DEAD_CACHED_POSE — Dead Cached Pose INFO HIGH

SaveCachedPose without matching UseCachedPose (or vice versa) — orphan node contributes no value.

How to Fix:
1. Navigate to the cached-pose node
2. Either delete the orphan Save (if the cache was retired) or add a matching Use (if you meant to consume it)
3. Compile — the dead node should be gone from the graph
Detection: Collects all SaveCachedPose and UseCachedPose names; flags any name present in one set but not the other. HIGH confidence.
#31 EMPTY_SM — Empty State Machine ERROR HIGH

State machine has zero states or the Entry node has no outbound linked transition — evaluation freezes on entry.

How to Fix:
1. Navigate to the SM node and open its sub-graph
2. Add at least one state connected from the Entry node
3. Give the state an anim pose (connect a sequence or blend)
4. Re-run the scan to confirm
Detection: Opens each StateMachine sub-graph; counts UAnimStateNode entries and checks Entry-node outbound links. HIGH confidence.
#32 BLEND_WT_SUM — Blend Weights Don't Sum to 1 WARN MED Auto-fixable

LayeredBoneBlend BlendWeights sum is outside [0.95, 1.05] — partial blending produces unexpected pose bias.

How to Fix:
AUTO-FIX: Normalizes weights so their sum equals 1.0, preserving the ratio between layers.

MANUAL: In the LayeredBoneBlend node, adjust individual BlendWeight defaults so their total is ~1.0.
Detection: Sums default BlendWeight pin values on each LayeredBoneBlend; flags when sum is outside [0.95, 1.05]. MEDIUM confidence.
#33 EMPTY_BRANCH_FILTER — LayeredBoneBlend Empty Bone Filter WARN HIGH

LayeredBoneBlend layer has no BranchFilters configured — blends the entire skeleton, defeating the purpose of the node.

How to Fix:
1. Navigate to the LayeredBoneBlend node
2. For each layer, set BranchFilters to the bones that layer should drive (e.g., spine_03 for upper body)
3. If no filter is needed, use a simple BlendTwoWay node instead
Detection: Iterates LayeredBoneBlend LayerSetup; flags any layer with an empty BranchFilters array. HIGH confidence.
#34 DISCONNECTED_SLOT_SRC — Slot Source Pin Disconnected WARN HIGH

Your character will T-pose between montages. Every time a montage (attack, reload, reaction) finishes, the character snaps into T-pose for a frame before returning to idle.

What's happening: A Slot node is where montages 'play over' your base animation. It has two pins:
  - Source (input, left): the base pose the slot mixes montages over
  - output (right): the final result after the montage plays

If Source has no connection, the slot has no base pose to fall back to when no montage is active — so it outputs T-pose.

Visual clue: The Slot node's left-side input pin (Source) has no white wire going into it. Node title shows the slot name, e.g. DefaultSlot.

How to Fix:
1. Click Navigate to open the AnimGraph
2. Find the flagged Slot node (yellow badge)
3. Look at its left input pin labeled Source
4. Find your base pose (typically the State Machine output or main BlendSpace — whatever was feeding the pose chain before the Slot was added)
5. Drag a wire from the base pose's output pin INTO the Slot's Source pin
6. Hit Compile
7. Test in-game: play a montage, let it finish — character should return to idle, not T-pose

Typical pose chain: State Machine -> Slot (DefaultSlot) -> Output Pose. Slot's Source input comes FROM the State Machine. Slot's output goes TO Output Pose.

If the slot isn't used: If no montage ever plays on this slot, you can delete the Slot node entirely — select it, press Delete, reconnect the State Machine output directly to the next node in your chain.
Detection: Reads each Slot node's Source input pin and flags disconnected states. HIGH confidence.
#35 ROOTMOTION_MODE_MISMATCH — RootMotion Mode Mismatch WARN HIGH

Your character won't move forward correctly — root motion from one or more animations is being silently dropped at runtime.

What's happening: Two settings have to agree: the AnimBP's Root Motion Mode (in Class Defaults) and each animation's Enable Root Motion flag (in the animation's asset settings). When Mode is RootMotionFromMontagesOnly, root motion only works for montages — SequencePlayer nodes silently drop it. When Mode is IgnoreRootMotion, the engine extracts the motion then throws it away — the character pivots in place.

Visual clue: Open the AnimBP > Class Defaults tab (top-right) > look for the Root Motion Mode dropdown. The flagged animation asset shows Enable Root Motion checked in its own Details panel.

Why it matters: The character moves in the preview but drifts or pivots in-game. Projectiles, dodge rolls, and root-motion attacks land in wrong positions. Ships silently — UE never warns about the mismatch.

How to Fix:
1. Click Navigate to open the AnimBP
2. Click the Class Defaults tab (top-right toolbar)
3. Find Root Motion Mode in the Details panel
4. Change it to Root Motion From Everything (this is the standard AAA locomotion setting — consumes root motion from both SequencePlayers and montages)
5. Alternative: If the animation shouldn't drive character movement, open the flagged AnimSequence asset and uncheck Enable Root Motion
6. Compile and test in-game — character should translate correctly

Why it's subtle: The AnimBP compiles clean. The animation plays fine in the viewport preview. The mismatch only shows as drift or in-place pivoting during PIE, and only for animations on non-montage nodes.
Detection: Reads RootMotionMode from AnimBP CDO. Walks SequencePlayer and BlendSpacePlayer nodes (SM-recursive), checks UAnimSequence::bEnableRootMotion against the active mode. HIGH confidence.
#36 LINKED_LAYER_NO_LAYER — LinkedAnimLayer No Layer Selected ERROR HIGH

Your LinkedAnimLayer node is missing a Layer selection. The character will T-pose in whatever part of the graph this node feeds.

What's happening: Linked Anim Layers are how UE5 routes parts of the AnimGraph to separate implementations — common for modular rigs (upper body vs lower body, vehicle vs on-foot states). The Layer field tells the node which layer interface to evaluate. When it's blank (None), the node has no pose chain to run, so it outputs the skeleton's default rest pose — T-pose — for every frame it's active.

Visual clue: Click the LinkedAnimLayer node, then look in the Details panel on the right. The Layer dropdown will be blank or show None. Nodes with this problem often have a yellow warning badge.

Why it matters: Any state machine state or graph path that flows through this node produces T-pose at runtime. The AnimBP compiles with zero errors.

How to Fix:
1. Click Navigate to open the AnimGraph
2. Click the flagged Linked Anim Layer node
3. In the Details panel, find the Layer dropdown
4. Select the correct layer from your Anim Layer Interface (e.g. Locomotion, UpperBody, FullBody)
5. If no interface exists yet, check whether you need a Linked Anim Graph node instead (which links entire AnimBPs rather than layer slots)
6. Hit Compile

Why it's subtle: The node looks identical whether or not a layer is selected — there's no red X or obvious visual indicator in the graph itself. You only discover the T-pose when you run the game and enter the affected state.
Detection: Walks UAnimGraphNode_LinkedAnimLayer nodes (SM-recursive). Flags any where FAnimNode_LinkedAnimLayer::Layer.IsNone(). HIGH confidence.
#37 CURVE_ALPHA_MISSING — Curve-Alpha References Missing Skeleton Curve WARN HIGH

A blend node's curve-driven alpha references a curve that was deleted (or never added) on the skeleton. The blend is permanently stuck at 0 — the layered animation never appears.

What's happening: Some blend nodes (e.g. Apply Mesh Space Additive, Blend List By Bool) can read their blend weight from a curve value instead of a variable. You set Alpha Input Type to Curve and name the curve in Alpha Curve Name. Every frame, the runtime looks up that name in the skeleton's curve metadata. If the curve doesn't exist there, it returns 0 — the blend never activates, no matter what the animation data says.

Visual clue: Click the flagged blend node > Details panel > look for Alpha Input Type set to Curve and Alpha Curve Name showing a name (e.g. AimWeight). The issue description will tell you the exact curve name that's missing.

Why it matters: The layered animation (aim offset, procedural lean, additive effect) simply never plays. The character animates fine for everything else, so it looks like an art problem rather than a rig issue. Very hard to spot without the check.

How to Fix:
1. Click Navigate to find the flagged blend node
2. Note the missing curve name from the issue description (e.g. AimWeight)
3. Option A — add the curve to the skeleton: Open the Skeleton asset > Window menu > Curves > click + > type the exact curve name > Save the Skeleton
4. Option B — switch to float input: On the blend node, change Alpha Input Type to Float and wire a variable or constant into the new Alpha pin instead
5. Compile and test — the blend should now activate

Why it's subtle: The AnimBP compiles cleanly. The node runs every frame. It just always reads 0. There's no in-editor warning, and the blend simply looks like it was never turned on.
Detection: Reflects over all anim graph nodes' inner Node structs. Finds AlphaInputType=Curve, reads AlphaCurveName, validates against USkeleton::GetCurveMetaData(). HIGH confidence.
#38 MONTAGE_SECTION_LOOP — AnimMontage Section Loops With No Exit WARN HIGH

A montage will play forever — it loops back to itself with no way out. Gameplay code expecting Montage_Ended won't receive the event.

What's happening: AnimMontages are divided into named Composite Sections (e.g. Start, Loop, End). Each section has a Next Section field that chains them together. When Next Section is blank (None), the montage ends — that's normal. But when a section points back to itself, or a chain forms a closed cycle (A → B → A) with no None exit, the montage is infinite. It will run until Montage_Stop is called explicitly.

Visual clue: Open the montage asset. At the bottom of the editor you'll see the Sections panel — a row of named sections with arrows between them. A self-loop shows as a section with an arrow curving back to itself. The issue description names the exact section and montage.

Why it matters: Gameplay code (reload complete, attack finish, cinematic hand-off) listens for Montage_Ended. That event never fires. The character locks into the montage permanently. In multiplayer, this can desync the entire animation state.

How to Fix:
1. Click Navigate — this opens the montage asset directly
2. In the Sections panel at the bottom, find the looping section named in the issue description
3. Click the section to select it
4. In its Details panel (or the Sections panel's Next field), set Next Section Name to None (montage ends here) or to a real exit section (e.g. End)
5. Save the montage

If the loop is intentional (e.g. a weapon idle loop that waits for a Montage_Stop from gameplay code): the check will still fire. Consider naming the section clearly (e.g. Loop_RequiresStop) so the team knows it's deliberate.

Why it's subtle: Sections panel arrows can be hard to read at a glance. The montage looks fine in the editor preview — it loops smoothly. The broken behavior only appears when Montage_Ended never fires in game.
Detection: Walks K2Node_CallFunction nodes calling Montage_Play / PlayMontage / PlayAnimMontage. Reads the literal Montage pin DefaultObject, dedupes, then detects self-loops and closed cycles via chain walk. HIGH confidence.
#39 BLENDSPACE_ZERO_AXIS — BlendSpace Axis Has Zero Range WARN HIGH

Your BlendSpace can't actually blend on one of its axes — the Min and Max values are the same, so there's no range to interpolate across.

What's happening: A BlendSpace uses axes to define its input range (e.g. Horizontal Axis: Speed 0..600, Vertical Axis: Direction -180..180). You place sample animations at positions within that range and UE interpolates between them based on runtime input values. If Min == Max on any axis, the range collapses to a single point — every input value maps to the same position, and no blending can happen along that dimension.

Visual clue: Open the flagged BlendSpace asset > Details panel on the right > expand Horizontal Axis (and Vertical Axis for 2D BlendSpaces) > look for Min and Max fields showing the same number (commonly both 0.0). The issue description tells you which axis is broken and its current Min/Max values.

Why it matters: Locomotion feels snapped or broken — speed changes or direction changes produce no visual blend. The character may always play the same animation regardless of input. If the zero-range axis is Speed, your character runs and walks at the same animation rate.

How to Fix:
1. Click Navigate to open the BlendSpace asset
2. In the Details panel, expand Horizontal Axis (and Vertical Axis if this is a 2D BlendSpace)
3. Set Min and Max to a sensible range for your game (e.g. Speed: Min 0, Max 600)
4. Re-place your animation samples across the new range in the BlendSpace grid
5. Save the BlendSpace

Common cause: The axis was set up but Max was never changed from its default of 0, or parameters were copy-pasted from another BlendSpace and one axis wasn't updated.

Why it's subtle: The BlendSpace saves and cooks without errors. In the BlendSpace editor preview the grid looks populated. The bug only shows as stiff, non-blending animation in-game.
Detection: Walks BlendSpacePlayer nodes (SM-recursive). Reads UBlendSpace::GetBlendParameter(0) and GetBlendParameter(1), flags axes where FMath::IsNearlyEqual(Min, Max). HIGH confidence.

4.2 General Blueprint Checks (13-26)

#13 BP_BROKEN_REF — Broken Asset Reference ERROR HIGH

Blueprint references an asset path that does not exist on disk.

How to Fix:
1. Navigate and look for red 'Missing' errors
2. Replace with a valid asset reference, or delete the node
3. Check Content Browser > Show Redirectors for moved assets
#14 BP_COMPLEXITY — Excessive Complexity WARN MED

Blueprint has 100+ unique nodes.

How to Fix:
1. Collapse related nodes to Functions
2. Move heavy logic to C++
3. Split responsibilities: one BP per concern
#15 BP_EMPTY_GRAPH — Empty Event Graph INFO HIGH

Blueprint exists but contains no meaningful logic.

How to Fix:
1. Check if it's a data-only Blueprint (might be intentional)
2. If truly empty: delete it in Content Browser
3. Fix Up Redirectors afterwards
#16 BP_TICK_HEAVY — Tick Performance Risk WARN MED

Event Tick enabled with high node complexity.

How to Fix:
1. Move non-essential logic to Timers (0.1-0.5s intervals)
2. Use event-driven patterns (overlaps, delegates)
3. Uncheck 'Start with Tick Enabled' if Tick isn't needed
#17 BP_SELF_CAST — Self-Cast INFO HIGH Auto-fixable

Blueprint casts to its own class type.

How to Fix:
AUTO-FIX: Removes the Cast node and reroutes connections through the source pin.

MANUAL: Delete the Cast node and use a 'Self' reference instead.
#18 BP_DEPRECATED_FUNC — Deprecated API WARN MED

Blueprint uses deprecated functions.

How to Fix:
1. Hover the node for the deprecation message
2. Delete and replace with the suggested alternative
3. Reconnect pins and test
#19 BP_CIRCULAR_DEP — Circular Dependency WARN MED

Two Blueprints reference each other.

How to Fix:
1. Create a Blueprint Interface
2. Move shared function signatures to the Interface
3. Replace Cast nodes with Interface Message calls
#20 BP_MASSIVE_ASSET — Oversized Asset WARN HIGH

Blueprint file is larger than 5MB.

How to Fix:
1. Check for embedded data (large arrays, mesh data)
2. Move data to separate Data Assets or Data Tables
3. Right-click > Size Map to see what's consuming space
#21 BP_HARD_REF — Hard Reference Bloat WARN MED

Excessive hard references via Cast nodes.

How to Fix:
1. Replace casts with Blueprint Interface calls
2. Use Soft Object References + Async Load
3. Use Reference Viewer to visualize the chain
#22 BP_EXPENSIVE_TICK — Expensive Tick Operations WARN MED

Expensive queries alongside Event Tick.

How to Fix:
1. Cache query results in BeginPlay
2. Use Timers instead of per-frame calls
3. For actor finding: use Actor Tags (faster)
#23 BP_DEBUG_NODES — Debug Nodes WARN HIGH Auto-fixable

PrintString or DrawDebug nodes still present.

How to Fix:
AUTO-FIX: Deletes all debug nodes and reroutes execution pins.

MANUAL: Search for PrintString/DrawDebug, delete, and reconnect white exec pins.
#24 BP_CONSTRUCT_HEAVY — Construction Script Misuse WARN HIGH

Heavy operations in Construction Script.

How to Fix:
1. Move SpawnActor/queries from Construction Script to BeginPlay
2. Use ChildActorComponents for editor-visible spawning
3. Keep Construction Script lightweight
#25 BP_FOREACH_PERF — ForEach Loop Performance INFO MED

ForEachLoop re-evaluates pure inputs every iteration.

How to Fix:
1. Cache the array input in a local variable before the loop
2. Connect the variable to ForEachLoop instead of the function call
#26 BP_TIMELINE_HEAVY — Excessive Timelines INFO MED

Multiple Timeline nodes creating hidden tick overhead.

How to Fix:
1. Merge Timelines with similar curves
2. Use FInterpTo for simple lerps
3. Use Set Timer for one-shot animations

5. Managing Checks Back to TOC

5.1 Enable / Disable Checks

Click Checks in the toolbar to open the Check Library. Each check has a checkbox — uncheck to disable it. Disabled checks are skipped during scanning. Click Save to persist your choices.

5.2 Suppression

To hide a specific issue you want to ignore:

  1. Select the issue in the results list
  2. Click Suppress
  3. The issue disappears from normal results

To view and manage suppressed issues:

  1. Check the Suppressed (N) checkbox in the filter bar
  2. The list now shows ONLY suppressed items
  3. Select any item and click Unsuppress to restore it

Suppressions persist across sessions via Settings.json. Use Settings > Clear All Suppressions to reset.

6. Custom Rules Back to TOC

Create project-specific diagnostic rules that run alongside the 39 built-in checks. No JSON editing required — use the visual editor.

Getting Started with Custom Rules

Three ways to create custom rules (pick whichever is easiest):

  1. Visual Editor: Settings > Edit Rules... — add rules via form fields, save with one click
  2. Create Example: Settings > Create Example — generates a starter file you can edit
  3. Import: Settings > Import... — load a rules file from a teammate or another project

Custom rules are stored in Saved/BPDoctor/CustomRules.json and loaded automatically on each scan.

Visual Rules Editor

Open Settings > Edit Rules... to see the editor. It shows all current rules and lets you:

  • View existing rules with their code, name, type, match string, and severity
  • Add new rules by filling in the form: Name, Code, Type (dropdown), Severity (dropdown), Match string, Description
  • Delete rules you no longer need
  • Save everything back to JSON with one click

10. FAQ & Troubleshooting Back to TOC

Q: The scan found issues but Auto-fixable shows 0. Why?

7 checks support auto-fix: BROKEN_BLEND_WT, MISSING_SLOT, ORPHANED_NODE, DUP_SLOT, BP_SELF_CAST, BP_DEBUG_NODES, BLEND_WT_SUM. The other issues require manual fixes. Click Navigate to jump to the exact node and follow the fix instructions in the detail panel.

Q: A check flagged something that isn't actually a problem. What do I do?

Click Suppress to hide it. If the check consistently produces false positives for your project, disable it via Checks > uncheck it > Save. Check the confidence level — LOW confidence checks are heuristic-based and expected to have some false positives.

Q: Can I add my own checks?

Yes. Create Saved/BPDoctor/CustomRules.json with your rules. See Section 6 for the format and examples. Rules are loaded automatically on each scan.

Q: How do I use BP Doctor in my CI/CD pipeline?

Run UnrealEditor-Cmd.exe YourProject.uproject -run=BPDoctor with format/output/severity flags. By default it returns exit code 0 even when issues are found — pass -failOnError (or -fail-on=error / -fail-on=warning) to make CI gate on severity. Supports SARIF 2.1.0 export via -sarif=PATH for GitHub Code Scanning, GitLab Security, and Azure DevOps. See the README's "Commandlet — Headless CI/CD" section for the full flag list.

Q: Does BP Doctor modify my Blueprints permanently?

No. Fixes only modify in-memory state (marked as dirty). You must explicitly Save the Blueprint to persist changes. The Revert button restores from a backup taken before the fix. If you close the editor without saving, all fixes are discarded.

Q: Which UE5 versions are supported?

UE 5.3 through 5.7. The plugin uses UE_VERSION_OLDER_THAN guards for API drift between versions.
